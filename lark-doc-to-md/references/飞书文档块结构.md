---

使用方式

1. 先拿到 `block_type`，直接跳到上面树中对应分支；
2. 该分支下出现的就是「一定存在」的字段，未写的都是「不会存在」；
3. 凡是带 `*` 的数组，实际长度 ≥0，遍历时按索引继续递归即可；
4. `text_element` 内部是**互斥 union**，只能有一个 key（text_run / mention_user / …），解析时先判 key 再取子结构。

把这棵树存成书签，再也不用在一万多字的平铺 JSON 里来回翻层级。
